# Photo Album Skill

## 这个 Skill 做什么

将一个文件夹里的照片 + 可选文字素材，生成一份**单文件 HTML 画册**，像一本可以翻阅的纸质摄影集。

**核心原则**：你是你生活和照片的唯一主体。AI 的文字不能参与感过强，不能有幻觉。AI 只做排版、分类、引导，不做主体表达。

## 何时使用

- 用户想把照片整理成一本画册
- 用户提到"照片书"、"photo book"、"整理照片"
- 用户有一个文件夹，里面有照片，想变成可翻阅的网页
- 用户想把旅行/人物/街拍/活动照片做成合集

## 三种模式

### 模式 1：杂志模式（Magazine Mode）
- 用户指定主题、时间
- 只操作一个文件夹
- 用户控制叙事线，AI 辅助排版
- 适合：有明确主题的拍摄（一次自驾游、孩子周岁）

### 模式 2：AI 全权模式（Auto Mode）
- AI 自动处理所有图片
- 自动分类、排序、生成文字
- 适合：懒人模式，"帮我把这堆照片做成画册"

### 模式 3：预设分类模式（Category Mode）
- 预设类型：出去玩（自驾/旅游）、记录人物（家人/朋友/宠物）、街拍、参加活动/探店
- 用户选择类型后，AI 按该类型的叙事模板处理
- 适合：没有明确主题，但知道这堆照片属于什么场景

## 工作流

### Step 0 · 确定模式（必须先问）

如果用户没有明确说用哪个模式，问：

> 你想怎么整理这些照片？
> 1. 杂志模式 —— 你告诉我主题和时间，我帮你排版
> 2. AI 全权 —— 我自动分类处理所有照片
> 3. 预设类型 —— 选一个场景（出去玩 / 记录人物 / 街拍 / 活动探店）

同时确认：
- 文件夹路径
- 是否有附带的文字素材（txt / md / 文档）
- 想要哪种输出风格（见下方"视觉风格"）

### Step 1 · 扫描与分析

1. 扫描文件夹，列出所有图片文件（jpg/png/webp/heic）
2. 读取任何文字素材（txt/md/doc）
3. 分析每张图片：
   - 内容描述（人物/风景/物体/场景）
   - 情绪基调（温暖/安静/活力/压抑）
   - 构图特点（特写/远景/对称/留白）
   - EXIF 信息（如果有）：时间、地点、相机
4. 按时间或逻辑排序

### Step 2 · 选图与叙事

基于狂阿弥画册经验：
- 600 张 → 选 70 张 → 实际用约 60 张
- 给后期排版留冗余（多选几张）
- 排序比排版更重要——要有逻辑线

叙事线模板：
```
开场（1-2 张）：定调，建立氛围
主体（分段）：按时间/空间/情绪分组
高潮（1-2 张）：最好的照片，最大篇幅
收束（1-2 张）：余韵，留白
```

### Step 3 · 排版

使用下方"页面布局库"中的模板，按叙事线组合页面。

排版技巧（来自狂阿弥）：
- **并置**：左右放两张不相关的照片，产生新意义
- **旋转**：竖构图变横放，打破方向感
- **前后呼应**：前页远景 → 后页特写
- **相似元素串联**：圆形 → 圆形，自然过渡
- **大小节奏**：照片一大一小交替，视觉有起伏

### Step 4 · 生成 HTML

使用 `assets/template.html` 作为基础，替换 SLIDES_HERE 注释处为生成的页面。

关键规则：
- 图片用 base64 内嵌（确保单文件）
- 所有文字基于用户素材或用户自己的描述，AI 不编造体验
- 文字引导：当用户不知道写什么时，提供问题引导（见"文字引导系统"）
- 照片是主体，文字是辅助

### Step 5 · 输出

- 默认输出为单 HTML 文件
- 可选：提示用户用浏览器打印功能转 PDF
- 可选：提供打印尺寸建议（A4/A5/方形）

## 提示词配置

所有可调整的提示词集中在 `references/prompts.md`，包括：
- 模式选择引导
- 图片分析提示
- 选图与排序提示
- 文字引导（通用 + 按场景分类）
- 页面标题生成
- 尾页文字
- AI 评分（可选）
- 排版决策

**需要调整提示词时，直接编辑 `references/prompts.md`。**

## 视觉风格

### 风格 A · 温暖杂志（默认）
- 衬线标题 + 非衬线正文
- 暖色调：米白底 + 深棕字
- 适合：人物、旅行、日常记录
- 主题色：`references/themes.md` 中的"暖墨"

### 风格 B · 冷静极简
- 无衬线为主
- 冷色调：灰白底 + 深灰字
- 适合：街拍、建筑、艺术摄影
- 主题色：`references/themes.md` 中的"银盐"

## 页面布局库

见 `references/layouts.md`，包含以下页面类型：
- L01 封面
- L02 全屏照片
- L03 双页并置
- L04 左文右图
- L05 图片网格（2x2 / 3x3）
- L06 大字引言
- L07 章节分隔
- L08 照片+说明
- L09 时间线
- L10 尾页

## 设计原则

1. **照片是主体**：文字永远不能喧宾夺主
2. **留白即呼吸**：不要把页面塞满
3. **节奏感**：大图 → 小图 → 文字 → 大图，像音乐
4. **真实性**：AI 不编造体验，只引导用户表达
5. **可打印**：布局考虑打印友好性
