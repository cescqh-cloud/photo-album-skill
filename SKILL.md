# Photo Album Skill

把一批照片做成真正愿意翻完的画册。默认先产出，再微调；不要把用户拦在模式选择题前。

## 默认体验

只把“照片目录”视为必需信息。标题、风格、文字和排序都可自动推断。

当路径已知时：

1. 直接扫描照片和同目录文字素材。
2. 采用“日常模式”生成第一版。
3. 打开成品检查首屏、中段、尾页和手机宽度。
4. 告诉用户成品路径，以及可以继续调整的 2–3 个方向。

只有以下情况才提问：

- 没有照片目录，且无法从当前工作区判断。
- 多个目录都像素材目录，选择会明显改变结果。
- 用户明确要求精确叙事，但缺少必要事实。

一次最多问一个关键问题。风格未指定时使用“暖日常”，不要先要求用户三选一。

## 先决定交付物

根据用户意图输出一种或两种：

- `album.html`：可离线翻阅、全屏和打印。
- 社交图片组：每页一张 JPG/PNG，带两位序号和 `upload-order.txt`。

用户提到小红书、朋友圈、发帖、配图或九宫格时，默认同时保留 HTML，并继续导出图片组。

## 两条编排路径

### 快速日常版（默认）

适合“帮我把这些照片做个相册”“先出一版看看”。

运行零依赖生成器：

```powershell
python scripts/build_album.py "照片目录" -o "album.html"
```

常用参数：

```powershell
python scripts/build_album.py "照片目录" -o "album.html" --title "周末散步" --theme warm
python scripts/build_album.py "照片目录" -o "album.html" --theme silver --sort name
python scripts/build_album.py "照片目录" -o "album.html" --max-photos 60 --recursive
python scripts/build_album.py "照片目录" -o "album.html" --composition editorial
python scripts/build_album.py "照片目录" -o "album.html" --composition gallery
```

生成器会：

- 读取 JPG、JPEG、PNG、WEBP、GIF。
- 优先按拍摄时间排序；读取不到时按文件名自然排序。
- 根据横竖比例、文字长度和前后照片自动组合页面。
- 将图片内嵌为 data URI，输出单个离线 HTML。
- 自动加入翻页按钮、键盘导航、进度、主题切换、图片放大和打印样式。

如环境安装 Pillow，生成器会自动纠正 EXIF 方向并压缩大图；没有 Pillow 也可正常生成，只是文件更大。HEIC/HEIF 需先转换为浏览器支持的格式。

### 图文与图片精排版（按需）

用户要求“图文排版”“认真选图”“图片排列”“杂志感”“做纪念册”时使用。

1. 批量查看照片，识别时间、场景、远近景、人物和重复镜头。
2. 删除明显失焦、误触和高度重复的照片；不要仅按技术分数淘汰有纪念意义的照片。
3. 先确定每页的信息层级：
   - 主图：承担第一视觉焦点，只选一张。
   - 辅图：补充动作、环境或细节，不与主图争夺面积。
   - 文字：解释照片之间的关系，不重复描述画面。
4. 根据素材选择结构，而不是按模板循环：
   - 有真实文字：`story` 或 `photo-note`。
   - 两张有动作连续/视觉呼应：`duo`。
   - 一张环境主图 + 两张细节：`triptych`。
   - 四张同场景碎片：`grid`。
   - 竖图、合照、高潮或停顿：`focus`。
5. 使用 4 段节奏：
   - 开场：1 张建立气氛。
   - 展开：用双图与网格交代人物、环境和细节。
   - 重音：1–2 张强图用单图大页面。
   - 收束：安静照片或用户原话，留出余韵。
6. 先用生成器产出可靠底稿，再按 `references/layouts.md` 精调页面。

`--composition editorial` 偏图文关系与留白；`--composition gallery` 偏图片组合；`auto` 根据文字密度自动选择。

## 导出小红书或朋友圈图片

生成 HTML 后运行：

```powershell
# 小红书：3:4，1080×1440
node scripts/export_social.mjs "album.html" --preset xhs --out "social\xhs"

# 微信朋友圈：1:1，1080×1080
node scripts/export_social.mjs "album.html" --preset moments --out "social\moments"
```

导出目录必须包含：

- `01-cover.png`、`02-*.png` 等按顺序命名的图片。
- `upload-order.txt`：用户照着序号上传。
- `manifest.json`：记录画幅、尺寸和全部文件。

小红书默认使用统一的 3:4 竖版画幅。朋友圈默认使用方图，减少九宫格预览裁切；用户更想保留竖版图文时使用 `--preset portrait`。详细规则见 `references/social-export.md`。

默认 PNG 导出只需要系统已有的 Edge、Chrome 或 Chromium。用户明确需要体积更小的 JPG 时，安装 `playwright-core` 并加 `--format jpg`。

## 素材与文字

- 扫描图片：`.jpg`、`.jpeg`、`.png`、`.webp`、`.gif`；遇到 HEIC/HEIF 时提示转换。
- 读取同目录的 `.txt`、`.md` 和用户明确提供的文档。
- 优先使用 EXIF 时间；没有时间时保持自然文件名顺序。
- 不移动、不重命名、不改写原照片。
- 默认输出到当前工作区，除非用户指定其他位置。

文字分三档处理：

1. 用户有文字：轻量校对并尽量保留原口吻。
2. 用户没有文字：只使用日期、地点、文件夹名和画面可见事实。
3. 需要情绪文字：先问一个具体问题，再根据回答整理；不得编造经历、地点、人物关系或感受。

标题要像相册标签，不像命题作文。优先使用“时间 + 地点/动作/人物”，避免“美好时光”“难忘旅程”“精彩瞬间”等空话。

## 视觉选择

默认主题是 `warm`（暖日常）：

- 柔和米白、墨色文字、轻微纸张质感。
- 适合手机照片、家人、朋友、宠物、旅行和普通生活。

根据素材自动切换：

- `silver`：街拍、建筑、黑白、冷色照片。
- `night`：夜景、演出、聚会、霓虹。

主题变量见 `references/themes.md`。布局片段见 `references/layouts.md`。只有精排或修改模板时才读取这些文件。

## 质量规则

- 照片优先，装饰退后；一页只保留一个视觉重心。
- 排版先建立主次，再考虑装饰；不要把所有照片做成同样大小。
- 文字和图片必须形成关系：解释前后、补充细节或保留用户原话。
- 保留少量“不完美但有记忆”的照片，日常画册不应像摄影比赛。
- 连续页面不要重复同一布局；单图 → 双图/网格 → 单图形成呼吸。
- 人像和竖图优先完整展示，不默认裁掉脸和身体。
- 没有真实文字时宁可留白，不生成假感悟。
- 外部字体不是必需依赖；成品离线打开也应保持可读。
- 单文件过大时先减少照片或使用 Pillow 压缩，不牺牲原文件。

## 验收

输出前必须：

1. 确认 HTML 可打开且图片全部加载。
2. 检查封面、至少一个中间页面和尾页。
3. 检查约 390px 手机宽度：无横向溢出，按钮可点，文字不遮挡主体。
4. 测试方向键、空格、上一页/下一页、主题切换和图片放大。
5. 确认打印时控制条与弹层不出现。
6. 导出社交图片时检查：
   - 每张尺寸完全一致。
   - 首图标题位于安全区，没有贴边。
   - 所有图片按 `01`、`02`、`03` 顺序连续。
   - 图文页没有文字溢出，竖图与合照没有错误裁切。
7. 运行技能验证：

```powershell
python C:\Users\Cesc\.codex\skills\.system\skill-creator\scripts\quick_validate.py .
```
