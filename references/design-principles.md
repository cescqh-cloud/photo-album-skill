# 日常画册设计原则

## 目标

成品首先要让用户愿意打开、愿意发给朋友、过一阵还愿意再看。不要为了“设计感”把普通生活包装成广告大片。

## 选图顺序

按以下优先级判断：

1. 纪念意义：人物关系、动作、独特时刻。
2. 叙事作用：能交代时间、地点、转折或细节。
3. 画面差异：远景、中景、特写和留白是否齐全。
4. 技术质量：清晰度、曝光、构图。

轻微偏色、轻微模糊但有记忆价值的照片可以保留。误触、严重失焦、完全重复的连拍应删除。

## 排序

默认先按拍摄时间。只有明显更好时才改为：

- 空间线：家里 → 路上 → 目的地 → 回家。
- 人物线：独处 → 相遇 → 一起活动 → 告别。
- 情绪线：安静 → 热闹 → 高潮 → 平静。

不要为了“故事”打乱用户能够辨认的真实顺序。

## 页面节奏

使用小循环，而不是套固定的 20 页模板：

```text
单图重音 → 双图对话 → 三/四图碎片 → 单图停顿
```

- 连续不超过两页同一布局。
- 每 3–5 页出现一次大图。
- 网格页之后优先安排一张安静单图。
- 封面与尾页都要简单，给照片留出入口和出口。

## 裁切

- 横图可用于封面和大图。
- 竖图优先用暗底完整展示，避免硬裁成人脸特写。
- 合照不裁掉边缘人物。
- 不确定主体时使用 `object-fit: contain`。
- 用户要求保留原构图时，所有关键照片使用完整展示。

## 文字

- 用户原话优先于 AI 文案。
- 没有文字时使用日期、地点、文件夹名或照片中可见事物。
- 一页正文通常不超过 80 个汉字。
- 标题尽量 2–8 个字，具体、口语、可辨认。
- 删除“美好、难忘、治愈、奔赴、岁月静好、值得珍藏”等模板词。
- 不编造天气、气味、人物关系、旅行路线和内心感受。

## 日常感

日常感来自选择与停顿，不来自滤镜：

- 保留早餐、路牌、手、鞋、窗外、宠物回头等小照片。
- 不必每页都有标题。
- 不必每张照片都有说明。
- 允许相册里有安静、普通、甚至略笨拙的一页。

## 技术底线

- 离线打开时字体和布局仍可用。
- 手机宽度不横向滚动。
- 控制按钮不遮挡重要内容。
- 打印时隐藏控制条、进度和灯箱。
- 原照片只读，所有压缩都作用于 HTML 内嵌副本。
