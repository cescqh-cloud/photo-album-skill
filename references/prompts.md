# 提示词

只在需要分析、精排或补充文字时读取。快速日常版不必先进行完整问答。

## 默认开场

路径已知时不要提问，直接制作。路径未知时只问：

```text
把照片所在的文件夹路径发我，我先按“暖日常”做一版；标题和风格之后都能改。
```

## 批量分析

```text
分析这批照片，输出紧凑 JSON。每张照片记录：

基础信息：
- file：文件名
- visible：画面中可确认的人物/物体/场景，不推测身份
- shot：wide / medium / close / detail
- orientation：landscape / portrait / square
- light：bright / soft / dark / mixed
- duplicate_group：与其他照片高度重复时填组名，否则 null
- technical_issue：none / blur / overexposed / underexposed / accidental

人情味评估（核心，不以技术好坏为标准）：
- shutter_reason：根据画面、文件名、时间线或用户素材判断“当时为什么按下快门？”。能判断就写具体原因，不能判断填 null
- punctum：画面中是否有能像箭一样射中观者的细节？描述具体是什么（一个眼神、一只手、一个旧物件等），没有则填 null
- vernacular_authenticity：是否捕捉到了未加粉饰的、随性甚至略带笨拙的真实瞬间？high / medium / low
- existence_proof：是否承载了对时间流逝的眷恋，证明某种珍贵事物"此曾在"？high / medium / low
- social_bond：是否记录了人与人之间的羁绊与归属感？是否具备超越私人范畴的叙事张力，让观者看到共享的人性经验？high / medium / low
- identity_projection：是否映照出拍摄者独特的生命体验与情感体温？high / medium / low

不要生成抒情文案，不要推测地点、关系、心情和事件。人情味评估基于画面可确认的细节判断。
```

## 选图

```text
从 {total} 张照片中选出不超过 {target} 张。

以"人情味"为核心标准，忘掉构图法则和对焦是否锐利，依次评估：
1. 快门理由：当时为什么按下快门？这个理由是否具体、真实、能被照片或用户素材支撑？
2. 刺点共鸣：是否击中内心，唤醒某种不可名状的情感？
3. 凡俗真实：是否接纳了生活的不完美，保留了粗粝但鲜活的日常状态？
4. 岁月留证：是否温柔地对抗了时间的流逝，宣告了某种珍贵存在的"此曾在"？
5. 叙事与羁绊：是否见证并巩固了人与人之间的连接与爱？是否具备超越私人范畴的普遍共鸣？
6. 灵魂投射：是否映照出拍摄者一生的阅历、审美与深情？

去掉误触和完全重复；保留少量技术普通但有独特人物动作或生活细节的照片。
一张因为手抖而模糊、但定格了爱人最开怀大笑的快照，其意义远超完美但冰冷的风光明信片。
结果按真实时间顺序排列，并说明被删除的重复组，不给照片打竞赛式分数。
```

## 排版

```text
为选中的照片安排页面。可用布局：
- focus：单图完整展示，适合竖图、重音和停顿
- duo：两张存在动作、视线、色彩或远近关系的照片
- triptych：一张主图加两个细节
- grid：四张同场景碎片
- story：一张照片加 28–100 字用户原话
- photo-note：一张照片加 8–28 字短旁注
- bleed-note：主体位置明确且有留白的照片上放短文

规则：
1. 每页先指定 main_visual，再指定 supporting_images。
2. 说明照片之间的具体关系；说不清关系就不要放在同一页。
3. 不把三张图片平均三等分；使用一主两辅。
4. 单图 → 图片组合 → 图文页 → 单图形成呼吸。
5. 合照与竖图不要强裁。
6. 没有真实文字时不用 story、photo-note 或 bleed-note。
7. 封面与结尾保持简单。

输出 JSON：
- layout
- main_visual
- supporting_images
- relationship
- crop_mode: cover / contain
- text_source: user / metadata / none
```

## 社交图片检查

```text
把画册页面作为一组连续社交图片检查：
- 第一张在 3:4 和中央方形预览中是否仍有清晰焦点？
- 每张是否只有一个视觉重心？
- 标题和正文是否远离边缘与人脸？
- 连续两张的密度是否有变化？
- 去掉页码后，文件顺序是否仍可由 01、02、03 文件名确认？

只报告需要修改的页面和具体原因。
```

## 标题

```text
根据文件夹名、日期、用户原话和可见事实，给出 5 个标题。
要求：
- 2–8 个汉字，像相册标签或朋友口吻。
- 优先具体的时间、地点、人物称呼、动作或物件。
- 禁用：美好时光、难忘旅程、精彩瞬间、治愈、奔赴、岁月静好。
- 不推测未提供的信息。
```

## 只问一个问题

用户希望加入情绪或故事，但没有素材时，从最贴近照片的问题中只问一个：

- 这组照片里，当时为什么按下快门？
- 这组照片里，你最不想忘记的是哪个瞬间？
- 这一天你最常记起的一个小细节是什么？
- 如果只给未来的自己留一句话，你会写什么？
- 这个人/宠物有什么只有你们知道的小习惯？
- 路上哪一张照片最像当时真实的感觉？

把用户回答轻量整理成 20–80 字，保留口语，不添加新事实。

## 文案自检

```text
检查所有文案：
- 删除空泛形容词和广告腔。
- 每段文案是否回答或暗示了“为什么按下快门”？没有就改成事实旁注或留白。
- 标出任何无法从用户素材或画面确认的事实。
- 将过长句拆短。
- 能留白的页面不要硬加文字。
输出修订后的文案，不解释写作技巧。
```
